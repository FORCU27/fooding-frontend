# 다중 휴게시간(Break Times) 수정

## 🎯 수정 목표

휴게시간을 여러 개 추가할 수 있도록 하고, API에서 불러온 데이터가 UI에 정확히 표시되며, 저장 시 모든 구간이 누락 없이 전송되도록 수정.

---

## 📁 수정된 파일

### 1. `page.tsx`

**문제**: `handleFormSubmit`에서 `DAY_OF_WEEK.map()` + `.find()`를 사용하여 요일당 첫 번째 휴게시간만 전송됨.

**수정**:
```diff
- const dailyBreakTimes = DAY_OF_WEEK.map((day) => {
-   const breakItem = formData.dailyBreakTimes?.find((b) => b.dayOfWeek === day);
-   return { ...breakItem };
- });
+ const existingMaxId = Math.max(...(formData.dailyBreakTimes ?? []).map((b) => b.id ?? 0), 0);
+ let tempIdCounter = existingMaxId + 1;
+ const dailyBreakTimes = (formData.dailyBreakTimes ?? []).map((item) => ({
+   id: item.id ?? tempIdCounter++,
+   dayOfWeek: item.dayOfWeek,
+   breakStartTime: item.breakStartTime ?? null,
+   breakEndTime: item.breakEndTime ?? null,
+ }));
```

### 2. `OperatingHourForm.tsx`

**수정 사항**:
- `OperatingHoursFormValues` 인터페이스에 `dailyBreakTimes` 필드 추가 (영업시간과 분리)
- `previewDailyTimes` 계산 시 영업시간과 휴게시간을 합쳐서 미리보기 표시
- `onSubmit`에서 ID 보존 로직 추가
- 중복 키 에러 해결: `key={item.dayOfWeek}` → `key={\`${item.dayOfWeek}-${index}\`}`

---

### 3. `BusinessHourForm.tsx`

**수정 사항**:
- `useMemo`로 기본값 계산 안정화 (무한 루프 방지)
- `useRef`로 초기화 상태 추적 (`hasInitializedRef`)
- `breakTimesFlat` 생성 시 `flatMap` 사용하여 다중 구간 지원
- 새 항목에 원본 ID 매칭 로직 추가 (같은 요일 내 순서 기준)

---

### 4. `BusinessHours.tsx`

**수정 사항**:
- `handleAddHours`: 빈 값 대신 기본 시간(`15:00~17:00` 또는 이전 종료 시간) 주입
- `everydayHoursList` 초기화 시 null/undefined 방어 로직 추가
- 외부 props 동기화 시 안전한 배열 접근 (`?.join()`)

## 🔑 핵심 변경 요약

| 문제 | 원인 | 해결 |
|------|------|------|
| 첫 번째 휴게시간만 저장됨 | `.find()`가 첫 번째만 반환 | `.map()`으로 전체 유지 |
| 새 항목에 ID 없어서 400 에러 | API가 모든 항목에 ID 요구 | 임시 ID 자동 생성 |
| 무한 루프 발생 | `useEffect` 의존성 충돌 | `useRef`로 변경 감지 |
| 중복 키 에러 | 같은 요일에 여러 항목 | 인덱스 포함 키 사용 |
